1) Deterministic fuzz / property harness  (tests/fuzz.rs)
---------------------------------------------------------
Four properties, each over a fixed-seed pseudo-random stream (reproducible):

  a. parsers_never_panic_on_arbitrary_bytes
     Every decoder (frame, message, dict, shadow, cursor) is fed arbitrary bytes and
     must return Ok or a typed Err — never panic, over-read, or over-allocate.
  b. frame_and_full_chain_round_trips_hold
     decode(encode(payload)) == payload; and a single byte flipped anywhere in a
     valid frame is never silently accepted as a *different* valid frame.
  c. dict_and_shadow_are_invertible_over_random_data
     decompress(compress(x)) == x;  mask(mask(x,k),k) == x.
  d. cursor_and_message_round_trips_hold
     Writer/Reader and the AgentMessage codec round-trip any structured value.

Soak result (release build):
     AATP_FUZZ_ITERS=2000000  ->  8,000,000 total random inputs (2,000,000 x 4 properties)
     test result: ok. 4 passed; 0 failed.   (finished in 1.74s)
  Default `cargo test` runs 20,000 iterations/property; the seed is fixed so the run
  is reproducible and does not destabilise the mutation prober.

2) Single-bit corruption — every class fails the build
------------------------------------------------------
The mutation operator set does not mutate literals, so each class of data corruption
was reproduced by hand and confirmed to redden the suite:

  corruption                                    caught by
  --------------------------------------------  -------------------------------------
  CRC32C table entry / off-by-one offset        table fold-digest + round-trip tests
  SplitMix64 keystream constant (1-bit flip)    keystream known-answer test
  dictionary entry text ("system" -> ...)       dictionary snapshot test
  frame / message length boundary (len+/-1)     exact-boundary buffer tests
  comparison / boolean operator in state.on()   exhaustive 3x3 transition table test

  Every one of the above produced at least one failing test. No class of single-bit
  change to the wire-critical constants or bounds can pass a green build.

3) Fail-closed on hostile input
-------------------------------
Combined with (1a), this upgrades "never panics on hostile bytes" from an
audit-by-trace to an empirical result over millions of adversarial inputs.
